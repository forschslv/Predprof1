# Система заказа питания в столовой

Это веб-приложение для управления системой заказа питания в столовой, реализованное на основе OpenAPI спецификации.

## Структура проекта

- [index.html](file:///C:/Users/alexa/PycharmProjects/Predprof1/front123/index.html) - Главная страница с навигацией
- [cafeteria_styles.css](file:///C:/Users/alexa/PycharmProjects/Predprof1/front123/cafeteria_styles.css) - Стили приложения
- [cafeteria_script.js](file:///C:/Users/alexa/PycharmProjects/Predprof1/front123/cafeteria_script.js) - Основной скрипт приложения
- [dish_types.js](file:///C:/Users/alexa/PycharmProjects/Predprof1/front123/dish_types.js) - Определения типов блюд
- `/auth/` - Авторизация и регистрация пользователей
- `/common/` - Общие компоненты (меню, студенты)
- `/orders/` - Управление заказами
- `/admin/` - Административная панель

## Функциональность

### Для студентов:
- Регистрация и авторизация через email и код подтверждения
- Просмотр меню на модуль
- Создание заказов на неделю
- Просмотр истории заказов
- Подтверждение оплаты

### Для поваров:
- Управление глобальным меню (добавление, редактирование, удаление блюд)
- Формирование меню на модуль
- Загрузка меню из файлов

### Для администраторов:
- Управление статусами заказов
- Генерация отчетов

## API эндпоинты

Приложение использует следующие эндпоинты из OpenAPI спецификации:

- `/register` - Регистрация/вход пользователя
- `/verify-code` - Проверка кода подтверждения
- `/menu` - Получение глобального меню
- `/menu/dish` - Создание блюда
- `/menu/upload` - Загрузка меню из файла
- `/module-menu` - Управление меню на модуль
- `/orders` - Создание и получение заказов
- `/orders/{order_id}/pay` - Подтверждение оплаты
- `/admin/orders/{order_id}/status` - Обновление статуса заказа
- `/admin/reports/docx` - Генерация отчетов

## Установка и запуск

1. Убедитесь, что сервер API запущен на `http://localhost:8000`
2. Откройте [index.html](file:///C:/Users/alexa/PycharmProjects/Predprof1/front123/index.html) в браузере
