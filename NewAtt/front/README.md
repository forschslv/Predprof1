# Структура проекта системы заказа питания

## Организация файлов по темам

Проект теперь организован по следующим тематическим папкам:

### Папки:
- **[auth/](auth/)** - Функциональность авторизации (вход, регистрация)
- **[orders/](orders/)** - Функциональность заказов и работы с меню на модуль
- **[common/](common/)** - Общая функциональность (профиль студента, функции повара)
- **[admin/](admin/)** - Административная панель и управление заказами
- **Корень** - Общие файлы (стили, вспомогательные скрипты)

### Файлы:

#### Корневые файлы:
- **[cafeteria_styles.css](cafeteria_styles.css)** - Глобальные стили
- **[dish_types.js](dish_types.js)** - Утилиты для типов блюд
- **[app.js](app.js)** - Центральная логика приложения
- **[main.html](main.html)** - Главная страница-навигатор

#### Тематические файлы:
- **[auth/index.html](auth/index.html)** - Страница авторизации
- **[auth/auth.js](auth/auth.js)** - Логика авторизации
- **[orders/index.html](orders/index.html)** - Страница заказов для студентов
- **[orders/orders.js](orders/orders.js)** - Логика заказов
- **[common/index.html](common/index.html)** - Страница функций повара
- **[common/cook.js](common/cook.js)** - Логика управления меню
- **[common/student.js](common/student.js)** - Логика профиля студента
- **[admin/index.html](admin/index.html)** - Страница администрирования
- **[admin/admin.js](admin/admin.js)** - Логика администратора

## Навигация
- [main.html](main.html) - служит главной точкой входа с навигацией по всем разделам
- Каждая тематическая папка содержит свой собственный полноценный HTML-файл
- Все файлы связаны через относительные пути

## Структура упрощает:
- Поддержку кода
- Работу нескольких разработчиков
- Масштабируемость
- Тестирование отдельных компонентов
